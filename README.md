# Asset AI

Asset AI is a property-specific AI assistant for multifamily real estate. It answers questions for one active `property_code` at a time, combining MySQL rent-roll analytics, property-scoped Chroma retrieval over scraped website content, OpenAI tool calling, and Angular-rendered response components.

The prototype keeps the LLM in an orchestration role: it may decide which allow-listed tool to call, but the backend owns property scoping, SQL filters, retrieval filters, analytics calculations, and KPI/table/chart payloads.

## Architecture

```text
Angular UI
  property selector + model selector + safe components
        |
FastAPI /api/chat
        |
OpenAI tool calling
        |
backend allow-listed tool registry
        |
        +-- MySQL rent-roll analytics, filtered by active property_code
        +-- Chroma website retrieval, where={"property_code": active_property_code}
        |
deterministic backend components + concise answer + sources
```

Important guardrails:

- Active `property_code` comes from the request, not the LLM.
- The tool registry strips model-supplied `property_code` / `propertyCode`.
- Chat history is scoped by `session_id + property_code`.
- Retrieval chunks carry `property_code` metadata and are queried with a property filter.
- Structured analytics components are generated by the backend.
- Cross-property names/codes are blocked before LLM tool calling.

## Prerequisites

- Python 3.11+
- Node 20+
- Docker Desktop, if using local MySQL through Compose
- An OpenAI API key for chat, embeddings, and Chroma rebuilds

## MySQL Setup

Start MySQL:

```powershell
docker compose up -d mysql
```

The Compose file creates:

```text
database: las_assets
user: las_assets_user
password: las_assets_password
port: 3306
```

If you already have MySQL running locally, use equivalent credentials in `backend/.env`.
`localhost` works when the backend runs directly on the host machine. If the backend runs inside Docker, change the host to the MySQL service name or `host.docker.internal`, depending on your setup. If MySQL access fails because the app connects through `127.0.0.1` instead of `localhost`, either use `localhost` in the URL or add a matching MySQL grant for `'las_assets_user'@'127.0.0.1'`.

## Data Ingestion

Place local source files under `backend/data/`:

```text
backend/data/RentRoll_LeaseCharges_NamesRedacted.zip
backend/data/property_websites.json
```

The root-level endpoint rebuilds the prototype data from local files:

```powershell
Invoke-RestMethod -Method Post http://localhost:8000/ingest
```

`POST /ingest`:

- Clears app-owned tables: chat, website pages, rent-roll rows, rent-roll summary groups, charge summaries, future residents, snapshots, and properties.
- Ingests rent-roll files from `backend/data`.
- Upserts property metadata from `property_websites.json`.
- Recreates `website_pages` from local scraped text files under `backend/storage/scraped_sites`.
- Rebuilds the Chroma collection from `website_pages`.
- Returns counts, warnings, errors, and per-property summary.

The endpoint is idempotent: rerunning it should not duplicate snapshots, rows, website pages, or chunks.

## Website Scraping And Retrieval

To scrape configured website pages from `backend/data/property_websites.json`:

```powershell
cd backend
python -m app.scripts.scrape_websites
```

To rebuild Chroma from stored `website_pages` and local scraped text:

```powershell
cd backend
python -m app.scripts.build_vector_index
```

Retrieval is always scoped with `where={"property_code": property_code}`. If no confident chunks pass the similarity threshold, the assistant should say the information was not found in the scraped website sample instead of guessing.

## Charge-Code Transparency

Rent classification is allowlist-based to avoid false positives such as `CURRENT`.

Known rent-like codes:

- `RENT`
- `MARKET`
- `BASERENT`
- `RENTAFF`
- `RENTRETL`
- `CONRENT`

Other documented codes seen in the data:

- `PARKING`
- `AMENITY`
- `TRASH`
- `PETFEEM`
- `GARAGE`
- `STORAGE`
- `SUBSIDY`
- `MTM`
- `RNTPROF`
- `RETXEST`

Uncertain codes are not classified as rent unless explicitly added to the allowlist in `analytics_service.py`.

## Assumptions And Limitations

- Website scraping is a limited representative sample.
- Some websites may render important content through JavaScript that `requests`/BeautifulSoup may not capture.
- Some accounting/property variants share one website.
- Analytics are limited to available rent-roll fields and implemented backend tools.
- Balance trend is rent-roll balance trend, not true accounting delinquency aging.
- The prototype uses deterministic backend tools for structured metrics and LLMs for orchestration/explanation.
- `POST /ingest` rebuilds from local project files; live scraping remains a separate script.
- Chroma rebuild requires `OPENAI_API_KEY` because embeddings are generated through OpenAI.
